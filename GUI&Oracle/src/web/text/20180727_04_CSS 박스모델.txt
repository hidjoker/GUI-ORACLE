 + CSS 박스 모델 (display 속성)
  - 요소를 어떻게 표현할 것인지에 대한 속성

  - width, height : 너비, 높이
  - border : 테두리
  - padding, margin : 내부, 외부 여백

  - 위의 다섯가지 속성이 영향을 받는다



 + inline
  - 컨텐츠가 부모 요소의 너비를 초과하면 새 행으로 자동 줄바꿈한다
  - border가 같이 줄바꿈이 되므로 모양이 깨진것럼 보일 수 있다

  - width, height
	적용이 안됨
	컨텐츠영역의 크기는 내용물에 맞춰 결정된다

  - margin : left,right만 적용됨, top,bottom 마진이 없음

  - line-height : 높이 조절을 할 때 사용(줄 간격 속성)



 + block
  - 너비가 부모요소의 크기만큼(100%) 설정된다

  - 요소의 크기가 부모요소보다 커지면 넘친부분에 overflow 속성이
   적용된다

  ** overflow
	요소 내의 내용이 요소보다 클 때 넘치는 부분의 처리 방법

	overflow-x, overflow-y 각각 너비, 높이에 대한 처리 설정

	visible : 보여주기
	hidden : 잘라내기
	scroll : 상하, 좌우 스크롤바 생성
	auto : 내용이 넘쳤을 때만 스크롤바 생성

  - <h1>~<h6>, <p>, <blockquote>, <dt>, <address>, <caption>과
   같은 종결 블록 요소는 또 다른 블록 요소를 자식으로 가질 수 없다


  - width, height : 적용가능
	기본값은 auto
	width - 부모요소의 너비만큼
	height - 내부 요소의 크기만큼

  - margin은 외부요소의 margin과 영역을 공유한다

  - width가 설정된 블록요소에 margin: 0 auto; 설정을 하면
   부모 영역내에서 수평 가운데 정렬을 한다

  - margin은 외부요소와의 거리(간격)을 나타내는데 음수로 지정되면
   컨텐츠영역이 겹쳐진다



 + inline-block
  - 인라인, 블록 섞어놓은 듯한 설정

  - 블록박스처럼 width, height, boder, padding, margin 적용 가능

  - 외부 요소와의 관계가 인라인과 같음


  - 요소 자체에 대한 레이아웃은 블록
  - 요소 외부에 대한 레이아웃은 인라인

  - width, heigth : 설정 가능
	컨텐츠에 맞추려면 auto 설정
	부모요소에 맞추려면 100% 설정

  - margin-top, margin-bottom 설정 가능

  - margin left, margin right : 외부요소와의 간격 지정



 + table
  - <table> 레이아웃

  - table의 외부 레이아웃은 블록과 유사


  - table의 외부에 margin이 있고, padding이 없음

  - 내부 요소(셀)에는 margin이 없고, padding이 있음


  - table과 셀의 각각 테두리가 존재한다
  - 테두리 겹치지 않으려면 border-collapse: collapse 스타일 적용




 + position

  - display로 설정하는 게 아님

  - position 속성을 이용하여 display외에 추가적으로 레이아웃을
   설정하는 것

  - 위치에 대한 설정을 한다 -> 기존 위치를 벗어남

  - 크기 설정할 수 있다
  - 내용에 맞게 축소되도록 할 수도 있다

  - 다른 박스모델에 영향을 주지 않음


  - static : 위치 설정 (기본값)
  - absolute : 절대 위치 지정
	브라우저 화면의 상하좌우 벽면 기준
  - relative : 상대 위치 지정 (기본 배치되어야할 위치 기준으로)
  - fixed : 고정 위치

  - left, right, top, bottom을 추가로 적용해서 위치 설정

 + CSS, Cascading Style Sheets
  - 마크업 언어가 표현되는 방법(style)을 기술하는 언어

  ** 마크업 언어 : HTML, XHTML, XML, ...


  - HTML 태그의 스타일을 지정하는 언어



 + CSS를 HTML문서에 적용하는 방법
  1. 인라인(inline) 방식
	HTML태그의 style 속성(attribute)로 지정

	- 장점
		태그를 보면서 스타일 지정 가능
		간편하게 작성할 수 있다
	- 단점
		각각의 태그에 적용하고 있기 때문에 관리가 힘들다



  2. 임베디드(embedded) 방식
	<head>태그 내에 <style> 태그를 생성하고 내부에 작성

	- 장점
		HTML 문서 내의 모든 태그의 스타일 한번에 관리한다

	- 단점
		태그와 스타일이 떨어져있어 스타일을 예측하기 어렵다



  3. 별도의 CSS파일을 로드하는 방식

   - 별도의 파일로 CSS Style을 관리하고 HTML에서 로드하는 방식

   - 확장자를 주로 .css로 처리한다

   - 장점
	목적에 따라서 HTML과 CSS를 완벽히 분리할 수 있다

   - 단점
	파일이 많아질 수 있다



   3-1. @import 방식
	<style> 태그 내에서 사용

	여러 CSS파일을 @import해서 불러올 수 있다
	( 계층적으로 파일을 관리하는 것도 가능 )
	( css파일안에서 다른 css파일을 @import할 수 있다)

	** 계층적으로 CSS파일을 사용하는 건 하지말자!



   3-2. <link> 태그 방식
	<head>태그 내에 <link> 태그를 사용

	개별 CSS파일을 각각 불러오는 방식으로 사용한다
	( @import처럼 계층적으로 사용하지 않는다 )
